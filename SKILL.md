# 金蝶 PPT 生成 Skill

基于官方金蝶集团 2025/2026 版 PPT 模板，直接生成完整 `.pptx` 文件。官方背景图和 Logo 内嵌，零配置可用。

---

## 使用前必读（每次生成都要读）

**按顺序读取以下三个文件后再开始任何操作：**

1. `style-guide.md` — 品牌色、字体、精确坐标、Logo 规范、背景图
2. `layout-presets.md` — 12 种标准版式 + 6 种思维模型版式的完整 PptxGenJS 代码模板
3. `pptx-builder.md` — 技术构建流程、QA 规范、常见陷阱

---

## 完整四阶段工作流

```
第零阶段：思维模型识别【新增】
  0.1 → 扫描用户输入，检测内容结构信号
  0.2 → 置信度高 → 直接确定模型，写入后续大纲
  0.3 → 置信度低 → 标记为「标准版式」，不强行套模型

第一阶段：内容发现
  1.1 → 一次性提四个问题（ask_user_input 工具）
  1.2 → 图像评估（如有上传图片）
  1.3 → 生成大纲（每页标注思维模型版式）→ 用户确认

第二阶段：内容脚本
  2.1 → 一次输出全部页面内容脚本（含模型排版指令）
  2.2 → 用户审阅反馈 → 确认定稿

第三阶段：生成文件
  3.1 → 编写 Node.js 构建脚本（含思维模型色块代码）
  3.2 → 执行生成 → 转图 → 视觉 QA → 修复
  3.3 → 交付 .pptx 文件
```

## 第零阶段：思维模型识别引擎

### 0.1 信号扫描规则

在读取用户任何输入（文字、大纲、文档）后，**逐页/逐段**扫描以下信号关键词：

| 模型 | 高置信信号（直接使用） | 辅助信号（结合语境判断） |
|------|----------------------|------------------------|
| **金字塔/MECE** | "结论先行"、"三大支柱"、"核心主张+三点支撑"、内容呈现明显「1个总结论 + N个分论点 + 论据」层级 | 汇报、战略解读、融资路演、"总结一句话" |
| **PDCA** | 明确出现"计划/执行/检查/改进"、"Plan/Do/Check/Act"、"PDCA" | 复盘、述职、质量管理、"下一步改进" |
| **SWOT** | 明确出现"优势/劣势/机会/威胁"、"Strengths/Weaknesses/Opportunities/Threats"、"SWOT" | 生态策略、竞争分析、战略规划 |
| **黄金圈** | 明确出现"为什么/WHY/HOW/WHAT"三层、"黄金圈"、"Simon Sinek" | 产品发布、Partner大会演讲、品牌故事 |
| **5W1H** | 明确出现 Who/When/What/Where/Why/How 六要素中 ≥4 个 | 方案说明、项目计划、活动策划 |
| **SCQA** | 明确出现"场景/冲突/问题/解决"、"Situation/Complication/Question/Answer"、"SCQA" | 提案、客户大会、问题分析报告 |
| **IPD五看** | 明确出现"五看"、"看行业/看客户/看机会/看竞争/看自己"中 ≥3项、"IPD" | 战略发布、生态大会、市场分析 |

### 0.2 置信度判断与处理

```
置信度高（直接使用，无需询问）：
  → 内容中出现模型名称本身（PDCA、SWOT、SCQA等）
  → 或同时出现该模型 ≥3 个核心要素词
  处理：在大纲对应页标注「[PDCA循环版式]」，直接采用模型色块排版

置信度低（标准版式）：
  → 仅有1-2个模糊信号
  处理：使用标准版式，内容脚本中用普通色块/要点列表呈现
```

### 0.3 混用规则

- **同一 PPT 允许多模型混用**，每页独立识别，互不干扰
- 大纲中每页单独标注版式类型（可同时存在 SWOT页、PDCA页、标准要点页）
- 混用上限建议：单份 PPT 不超过 3 种思维模型版式，避免视觉割裂
- 若检测到 4 种以上，优先保留主要模型，其余降级为「标准版式」

## 第一阶段：内容发现

### 1.1 四个问题（一次性）

> **⚡ 快速模式**：若用户同时满足以下两条 → 跳过全部问题，直接进入 Phase 0 + 1.3（大纲+内容脚本合并一次输出，末尾一个确认问题）：
> 1. 已提供完整文字内容（文档/大纲/明确文案）
> 2. 说了"直接生成"/"不用确认"/"快速"/"省略确认"

在**一次** `ask_user_input` 调用中同时提出以下四个问题：

```
问题1  标题:"场景"  类型:single_select
  选项: 内部汇报 / 伙伴赋能 / 客户大会 / 方案提案

问题2  标题:"页数"  类型:single_select
  选项: 短（5-10页）/ 中（10-20页）/ 长（20页以上）

问题3  标题:"内容状态"  类型:single_select
  选项: 内容已完整准备 / 有草稿需整理 / 只有主题方向

问题4  标题:"素材"  类型:multi_select
  选项: 有图片（稍后上传）/ 有数据图表 / 无额外素材
```

**根据回答执行：**

| 回答 | 动作 |
|------|------|
| 内容状态 = 只有主题方向 | 追问：主题是什么？核心要传达的一句话？ |
| 素材 = 有图片 | 回复「请上传图片，支持 PNG/JPG/SVG/WEBP」，等待后进入 1.2 |
| 素材 = 有数据图表 | 回复「请提供数据（Excel/CSV/数值列表均可），我会生成图表插入对应页」，等待后进入 1.3 |
| 素材 = 无额外素材 | 直接进入 1.3 |
| 内容已完整且无素材 | 跳过 1.2，直接进入 1.3 |

**场景对默认结构的影响（记住此偏好，影响后续大纲）：**

| 场景 | 默认包含页面 | 偏好版式 |
|------|------------|---------|
| 内部汇报 | 内容页（无封面无结尾） | 数据卡片、要点列表、时间轴、**PDCA/金字塔** |
| 伙伴赋能 | 封面 + 内容页 + 结尾 | 架构图、流程图、对比页、**SWOT/黄金圈** |
| 客户大会 | 封面 + 目录 + 章节页 + 内容 + 结尾 | 数据卡片、**IPD五看**、图文并排、**黄金圈/SCQA** |
| 方案提案 | 封面 + 目录 + 内容页 | 对比页、流程图、数据卡片、**SCQA/5W1H** |

### 1.2 图像评估（用户上传图片后执行）

用户在对话中直接上传图片，Claude 读取后**逐张输出评估**：

```
📎 图片 1：[文件名]
   内容描述：[图片展示了什么]
   可用性：✅ 清晰可用 / ⚠️ 分辨率偏低，建议替换 / ❌ 不适合PPT
```

图片以 **base64 格式内嵌** 进 pptx（完全自包含）。进入 1.3 时，在大纲每页标注分配的图片编号。

## 第二阶段：内容脚本

### 2.1 一次输出全部页面内容脚本

思维模型页须额外注明模型结构，格式如下：

**思维模型页脚本格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】版式：[PDCA循环版式]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
标题：2025年度项目复盘

模型结构：
  P（计划）：年初制定的三大目标
  D（执行）：实际落地情况，完成率80%
  C（检查）：发现效率瓶颈在交付环节
  A（改进）：2026年引入自动化流程

排版指令：4格循环色块（顺时针），中央↻符号
图片：无
```

**IPD五看页脚本格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】版式：[IPD五看版式]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
标题：市场全景分析

模型结构：
  看行业：企业数字化加速，AI原生应用渗透率 <10%
  看客户：核心诉求从降本转向增效，愿为Agent买单
  看机会：Skill生态市场规模预计 2026年达2亿
  看竞争：友商缺乏元数据治理，在AI准确性上有短板
  看自己：苍穹PaaS底座扎实，ISV生态已达2800+

排版指令：5列等宽卡片横排，奇偶列顶部交替主蓝/品青，内容区浅灰蓝底
图片：无
```

**标准内容页脚本格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】版式：[内容-要点列表]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
标题：xxxxxx

正文内容：
• 普通要点（≤30字，18pt普通）
• ★ 重点要点（≤30字，加粗+蓝色/金色强调）
• 普通要点

图片：无 / 图片N → 位置：左侧 / 右侧 / 全页
排版建议：[三列卡片 / 横向流程 / 左右对比 / 纯要点]
```

**层级标注规则：**
- `★` = 重点强调（20pt 加粗，蓝色 `1770EA` 或黄色 `FFB800`）
- 无标记 = 普通内容（18pt 普通，1.3× 行距）
- `[底色块]` = 思维模型一级信息（品牌色背景 + 白字加粗）
- `[icon:📊]` = 指定图标（写在卡片/格子标注行，生成时调 `addIconBadge` 或直接 `addText`）

**图标使用规则（内容脚本阶段）：**
1. **思维模型页** — 每格默认带图标（见 `style-guide.md § 10.4`），脚本中可用 `[icon:默认]` 表示采用推荐值，或 `[icon:🎯]` 覆盖
2. **Bento 卡片页** — 主卡（深色背景）配大图标（28–32pt），次卡（浅色）配中图标（24pt）
3. **要点列表页** — 行首图标尽量统一（同一页不超过 2 种图标），或全部不用图标
4. **图标密度上限** — 单页最多 6 个图标；同一页不得出现 3 种以上图标类别（避免视觉噪声）
5. **颜色规则** — 深色背景格（主蓝/品青）→ 图标 `color: FFFFFF`；浅色底 → 图标 `color: 1770EA`

**内容原则：**
- 每页 ≤5 个要点，单条 ≤30 字
- 思维模型页每个格 ≤3 个要点，核心词 ≤6 字加粗置中
- 内容不超过版面 60%，保持视觉留白
- 品牌色背景上的文字必须用白色
- 语调简洁有力，企业高管演讲调性

### 2.2 用户确认

## 常见情况处理

| 情况 | 处理方式 |
|------|---------|
| 用户直接提供完整内容 | 跳过 1.1，从 Phase 0 扫描 + 1.3 大纲确认开始 |
| 用户明确指定模型 | 跳过 Phase 0 扫描，直接采用指定模型版式 |
| 用户说「不要思维模型」 | Phase 0 结果全部忽略，按标准版式处理 |
| 用户说「直接生成」/「不用确认」 | 触发快速模式：跳过 1.1 问题，Phase 0+1.3+2.1 合并输出，末尾一个确认问题 |
| 用户只要内容脚本不要文件 | 完成第二阶段后停止 |
| 用户说「不要封面/结尾」 | 记住偏好，大纲中删除对应页 |
| 章节超过4个 | 建议合并或设附录，目录页最多4章 |
| 单页内容过多 | 拆为2页，用承接标题「（一）」「（二）」 |
| **用户要求修改某页** | ① 定位该页在 `build_pptx.js` 中的函数调用；② 仅修改该页的函数参数或版式调用；③ 重新执行完整的 node→转PDF→转图→视觉QA 流程；④ 确认后交付新文件（不要只修改脚本而不重新转图） |
| 检测到4种以上模型 | 保留主要3种，其余降级为标准版式，大纲中注明 |
| 用户上传已有 .pptx | 先用 python-pptx 解析脚本（见 `pptx-builder.md §3`）提取内容，再走完整四阶段重排为金蝶风格 |
| 用户提供数据要求图表 | 收集数据后用 `pptx.addChart()` 插入（见 `pptx-builder.md §10`），在大纲标注「[图表页-折线/柱/饼]」 |
