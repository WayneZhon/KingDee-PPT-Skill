# 金蝶 PPT 生成 Skill

基于官方金蝶集团 2025/2026 版 PPT 模板，直接生成完整 `.pptx` 文件。官方背景图和 Logo 内嵌，零配置可用。

---

## 使用前必读（每次生成都要读）

**按顺序读取以下三个文件后再开始任何操作：**

1. `style-guide.md` — 品牌色、字体、精确坐标、Logo 规范、背景图
2. `layout-presets.md` — 12 种版式完整 PptxGenJS 代码模板
3. `pptx-builder.md` — 技术构建流程、QA 规范、常见陷阱

---

## 完整三阶段工作流

> **⚡ 进入前先判断模式：**
> - 用户上传了旧 `.pptx` 并希望优化 → **初稿优化模式**（见下方独立章节）
> - 其他所有情况 → **标准创作模式**（继续下方三阶段）

```
【标准创作模式】
第一阶段：内容发现
  1.1 → 一次性提五个问题（ask_user_input 工具）
  1.2 → 图像评估（如有上传图片）
  1.3 → 生成大纲 → 用户确认

第二阶段：内容脚本
  2.1 → 一次输出全部页面内容脚本
        （若用户选择「需要丰富内容」→ 每页同时给出原版 + 丰富版对比）
  2.2 → 用户审阅反馈 → 确认定稿

第三阶段：生成文件
  3.1 → 编写 Node.js 构建脚本
  3.2 → 执行生成 → 转图 → 视觉 QA → 修复
  3.3 → 交付 .pptx 文件
```

---

## 第一阶段：内容发现

### 1.1 五个问题（一次性）

在**一次** `ask_user_input` 调用中同时提出以下五个问题：

```
问题1  标题:"场景"  类型:single_select
  选项: 内部汇报 / 伙伴赋能 / 客户大会 / 方案提案

问题2  标题:"页数"  类型:single_select
  选项: 短（5-10页）/ 中（10-20页）/ 长（20页以上）

问题3  标题:"内容状态"  类型:single_select
  选项: 内容已完整准备 / 有草稿需整理 / 只有主题方向

问题4  标题:"图片"  类型:single_select
  选项: 有图片（稍后上传）/ 无图片

问题5  标题:"内容丰富度"  类型:single_select
  选项: 保持原始内容不扩展 / 帮我丰富及优化内容
```

**根据回答执行：**

| 回答 | 动作 |
|------|------|
| 内容状态 = 只有主题方向 | 追问：主题是什么？核心要传达的一句话？ |
| 图片 = 有图片 | 回复「请上传图片，支持 PNG/JPG/SVG/WEBP」，等待后进入 1.2 |
| 图片 = 无图片 | 直接进入 1.3 |
| 内容已完整且无图片 | 跳过 1.2，直接进入 1.3 |
| 内容丰富度 = 帮我丰富及优化内容 | **记住此偏好**，在第二阶段 2.1 中每页输出「原始版 vs 丰富版」对比 |

**场景对默认结构的影响（记住此偏好，影响后续大纲）：**

| 场景 | 默认包含页面 | 偏好版式 |
|------|------------|---------|
| 内部汇报 | 内容页（无封面无结尾） | 数据卡片、要点列表、时间轴 |
| 伙伴赋能 | 封面 + 内容页 + 结尾 | 架构图、流程图、对比页 |
| 客户大会 | 封面 + 目录 + 章节页 + 内容 + 结尾 | 数据卡片、IPD五看、图文并排 |
| 方案提案 | 封面 + 目录 + 内容页 | 对比页、流程图、数据卡片 |

---

### 1.2 图像评估（用户上传图片后执行）

用户在对话中直接上传图片，Claude 读取后**逐张输出评估**：

```
📎 图片 1：[文件名]
   内容描述：[图片展示了什么]
   可用性：✅ 清晰可用 / ⚠️ 分辨率偏低，建议替换 / ❌ 不适合PPT
   建议用途：[封面配图 / 产品截图专页 / 架构图复用 / 数据图表页]
   主色调：[描述]
```

**围绕图片与文字共同设计大纲（不是先规划再配图）：**
- 3 张产品截图 → 3 张图文并排内容页（版式 10）
- 流程图/架构图 → 直接复用（版式 08）
- 品牌/Logo 图 → 封面或结尾
- 数据图表 → 数据卡片页（版式 05）

图片以 **base64 格式内嵌** 进 pptx（完全自包含，发给任何人都能看到图片）。

进入 1.3 时，在大纲每页标注分配的图片编号。

---

### 1.3 生成大纲并确认

输出完整大纲：

```
──────────────────────────────────────────────────
  金蝶PPT大纲 · [主题] · 共 X 页
──────────────────────────────────────────────────
第 01 页  [封面页]        主标题 / 副标题 / 日期
第 02 页  [目录页]        4个章节导览
第 03 页  [章节分隔 01]   章节名 / 副标题
第 04 页  [内容-要点列表] 页面标题  ← 图片1（右侧）
第 05 页  [内容-数据卡片] 页面标题  3个指标卡片
第 06 页  [章节分隔 02]   章节名
...
第 XX 页  [结尾页]        多语言致谢
──────────────────────────────────────────────────
```

然后使用 `ask_user_input` 提问：

```
问题  标题:"大纲确认"  类型:single_select
  选项: 没问题，开始写内容 / 需要调整大纲 / 需要调整图片分配
```

- 选「没问题」→ 进入第二阶段
- 选「调整」→ 修改后再次确认，直到用户满意

## 第二阶段：内容脚本

### 2.1 一次输出全部页面内容脚本

**根据用户在 1.1 的选择，执行对应格式：**

---

**① 若用户选择「保持原始内容不扩展」**，按标准格式逐页输出：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】版式：[版式名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• ★ 重点要点（≤30字，加粗+蓝色/金色强调）

图片：无 / 图片N → 位置：左侧 / 右侧 / 全页
排版建议：[三列卡片 / 横向流程 / 左右对比 / 纯要点]

```

**② 若用户选择「帮我丰富及优化内容」**，每页同时给出两个版本供用户对比选择：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】版式：[版式名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

▷ 原始版（基于您提供的内容）
标题：xxxxxx
• 原始要点 A
• 原始要点 B
排版建议：[版式]

▶ 丰富版（Claude 优化扩展）
标题：xxxxxx（优化后标题）
• ★ 重点要点（补充数据/案例/金蝶背书）
• 扩展要点（补充上下文/业务逻辑）
• 普通要点
排版建议：[更丰富的版式，如增加图标/数据卡片]
丰富说明：[说明新增了什么内容及理由，1-2句]
```

全部页面输出完毕后，使用 `ask_user_input` 提问：

```
问题  标题:"内容版本确认"  类型:single_select
  选项: 全部使用丰富版 / 全部使用原始版 / 我来逐页告诉你选哪个
```

- 选「逐页告诉你」→ 用户逐条说明，Claude 记录每页选择后合并为定稿内容脚本

---

**层级标注规则（两种版本均适用）：**
- `★` = 重点强调（20pt 加粗，蓝色 `1770EA` 或金色 `FFC000`）
- 无标记 = 普通内容（18pt 普通，1.3× 行距）
- `[底色块]` = 架构图一级信息（深色背景+白字加粗）

**内容原则：**
- 每页 ≤5 个要点，单条 ≤30 字
- 内容不超过版面 60%，保持视觉留白
- 蓝色背景上的 Icon 必须用白色
- 语调简洁有力，企业高管演讲调性

### 2.2 用户确认

> 「以上是全部 X 页的内容脚本。如需修改，告诉我页码和修改内容（可多处），我统一调整后再生成文件。没有问题请回复「开始生成」。」

收到「开始生成」→ 进入第三阶段。

## 初稿优化模式（用户上传已有 .pptx）

当用户上传了现有 `.pptx` 文件并要求优化排版或美化时，切换为此模式，**不执行标准三阶段流程**。

### O.1 提取与解析原稿

```bash
# 用 python-pptx 提取原稿内容
pip install python-pptx --break-system-packages -q
python3 << 'EOF'
from pptx import Presentation
prs = Presentation("/mnt/user-data/uploads/[文件名].pptx")
for i, slide in enumerate(prs.slides):
    print(f"\n=== 第{i+1}页 ===")
    for shape in slide.shapes:
        if shape.has_text_frame:
            for para in shape.text_frame.paragraphs:
                text = para.text.strip()
                if text:
                    print(text)
EOF
```

提取完毕后，**向用户展示解析摘要**：

```
📄 已解析您的PPT原稿：共 X 页
──────────────────────────────
第1页：[标题] — [内容摘要，20字内]
第2页：[标题] — [内容摘要]
...
──────────────────────────────
✅ 识别到以下问题需要优化：
  • [问题1，如：第3页内容过密，建议拆页]
  • [问题2，如：多页版式重复，缺乏节奏变化]
  • [问题3，如：部分标题措辞不够精炼]
```

### O.2 确认优化范围

使用 `ask_user_input` 提问：

```
问题1  标题:"内容优化范围"  类型:single_select
  选项: 仅优化排版和视觉，文字内容不改 / 排版+全面优化措辞（需我确认后再生成）

问题2  标题:"页面结构"  类型:single_select
  选项: 保留原有页面结构和顺序 / 允许Claude调整页面顺序和拆合

问题3  标题:"图片"  类型:single_select
  选项: 保留原稿中的图片 / 不需要图片
```

### O.3 输出优化后内容脚本（需用户确认）

**若用户选择「排版+全面优化措辞」**，在生成文件前，逐页对比展示：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】建议版式：[版式名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

原稿内容：
  [原文照录]

优化后内容：
  标题：[优化标题]
  • ★ [精炼后要点]
  • [精炼后要点]
  优化说明：[1句说明改了什么、为什么]
```

全部页面展示完毕后，使用 `ask_user_input` 确认：

```
问题  标题:"优化稿确认"  类型:single_select
  选项: 没问题，开始生成 / 需要调整某几页 / 全部保留原文只改排版
```

**若用户选择「仅优化排版和视觉，文字内容不改」**，跳过内容对比，直接进入 O.4。

### O.4 生成文件

与标准模式第三阶段完全相同：读取 style-guide.md + layout-presets.md → 编写构建脚本 → 执行 → 视觉QA → 交付。

交付说明注明：「共 X 页，基于原稿优化，已通过视觉QA。」

## 常见情况处理

| 情况 | 处理方式 |
|------|---------|
| 用户上传了 .pptx 并要求优化 | 切换为**初稿优化模式**，不走标准三阶段 |
| 用户直接提供完整内容 | 跳过 1.1，从 1.3 大纲确认开始 |
| 用户只要内容脚本不要文件 | 完成第二阶段后停止 |
| 用户说「不要封面/结尾」 | 记住偏好，大纲中删除对应页 |
| 章节超过4个 | 建议合并或设附录，目录页最多4章 |
| 单页内容过多 | 拆为2页，用承接标题「（一）」「（二）」 |
| 用户要求修改某页 | 仅重新生成受影响的幻灯片，更新文件 |
| 内容丰富度选「逐页决定」 | 等用户说完所有页的选择，一次性合并定稿再进入第三阶段 |
